# GARVIS メモリ ライフサイクル ディレクティブ v1

**権限:** Adrien D. Thomas / ProCityHub  
**リポジトリ:** `~/GARVIS`  
**ブランチ:** `feature/memory-lifecycle-v1`

このディレクティブはローカルの証拠認識型メモリ ライフサイクルを導入し、続いて Git ワークフロー（validation、commit、push、pull request、CI ウォッチ、squash マージ、ブランチ削除、ローカル `main` 更新）を自動で実行します。

## メモリ設計

- 作業記憶、エピソード記憶、意味記憶、手続き記憶、コア記憶、残留トレース記憶。
- 保持は時間とともに減衰し、反復／検索で強化されます。
- 証拠ステータスは顕著性や感情とは分離されたままです。
- 関連想起はローカルの GGUF モデルに到達する前に制限されます。
- 古く弱い言語情報は小さな "ghost trace" に圧縮される場合があります。
- トレース記録はプロンプトからの想起対象外で、事実としては数えられません。
- コア／保護された記憶は自動的にトレース化されることはありません。
- 自動メンテナンスは決してハード削除を行いません。
- 明示的な忘却には `FORGET-<memory_id>` が必要です。

これは心理学に着想を得た機能的なソフトウェアのアナロジーです。生物学的シミュレーションではなく、意識を確立するものでもありません。

## ビルドの境界

このディレクティブは、いかなる検証エラーや GitHub での失敗でも停止します。モデル重み、資格情報、`.secrets`、秘密鍵、ローカル データベースをコミットすることは決してありません。メッセージ送信、取引、送金、コンテンツ公開を行うこともありません。
