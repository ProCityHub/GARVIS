# GARVIS メモリ ライフサイクル 指令 v1

**権限:** Adrien D. Thomas / ProCityHub  
**リポジトリ:** `~/GARVIS`  
**ブランチ:** `feature/memory-lifecycle-v1`

この指令は、ローカルで証拠志向のメモリ ライフサイクルを導入し、その後に Git ワークフロー（検証、コミット、プッシュ、プルリクエスト、CI 監視、squash merge、ブランチ削除、ローカル `main` 更新）を自動で実行します。

## メモリ設計

- ワーキング、エピソード、セマンティック、プロシージャル、コア、残渣トレースの各メモリ。
- 保持は時間とともに減衰し、反復/想起で強化されます。
- 証拠の状態は、顕著性や感情とは分離されたままです。
- 関連する想起は、ローカルの GGUF モデルに到達する前に制限されます。
- 弱い古い言語は小さな「ゴースト トレース」に圧縮される場合があります。
- トレース記録はプロンプト想起から除外され、事実として数えられません。
- コア/保護メモリは自動的にトレースされることはありません。
- 自動メンテナンスはハード削除を決して行いません。
- 明示的な忘却には `FORGET-<memory_id>` が必要です。

これは心理学に基づく機能的なソフトウェアのアナロジーです。生物学的シミュレーションではなく、意識を確立するものでもありません。

## ビルド境界

この指令は、いかなる検証または GitHub の失敗でも停止します。モデルの重み、認証情報、`.secrets`、秘密鍵、ローカル データベースをコミットすることは決してありません。メッセージ送信、取引、送金、コンテンツ公開を行うことも決してありません。
