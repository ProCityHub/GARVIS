# GARVIS メモリ ライフサイクル指令 v1

**責任者:** Adrien D. Thomas / ProCityHub  
**リポジトリ:** `~/GARVIS`  
**ブランチ:** `feature/memory-lifecycle-v1`

この指令は、ローカルで証拠に基づくメモリのライフサイクルを導入し、その後に Git のワークフロー（検証、コミット、プッシュ、プルリクエスト、CI の監視、squash マージ、ブランチ削除、ローカル `main` の更新）を自動で実行します。

## メモリ設計

- ワーキング、エピソード、セマンティック、手続き、コア、レジデュアルトレースの各メモリ。
- 保持は時間とともに減衰し、反復／想起によって強化されます。
- 証拠ステータスは顕著性や感情とは分離されます。
- 関連する想起は、ローカル GGUF モデルに到達する前に制限されます。
- 弱く古い言語情報は小さな「ゴーストトレース」に圧縮される場合があります。
- トレース記録はプロンプトでの想起から除外され、事実として数えることはできません。
- コア／保護メモリは自動でトレースされることはありません。
- 自動メンテナンスではハード削除を決して行いません。
- 明示的な忘却には `FORGET-<memory_id>` が必要です。

これは心理学に着想を得た機能的なソフトウェアのアナロジーです。生物学的シミュレーションではなく、意識を確立するものでもありません。

## ビルドの境界

この指令は、検証または GitHub の失敗が発生した場合に停止します。モデルの重み、資格情報、`.secrets`、秘密鍵、ローカルのデータベースをコミットすることは決してありません。メッセージの送信、取引、送金、コンテンツの公開は一切行いません。
