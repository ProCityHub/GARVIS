# GARVIS メモリ ライフサイクル 指令 v1

**Authority:** Adrien D. Thomas / ProCityHub  
**Repository:** `~/GARVIS`  
**Branch:** `feature/memory-lifecycle-v1`

この指令は、ローカルの証拠指向メモリ ライフサイクルを導入し、その後に Git ワークフロー（検証、コミット、プッシュ、プルリクエスト、CI 監視、スカッシュ マージ、ブランチ削除、ローカル `main` 更新）を自動で実行します。

## メモリ設計

- 作業、エピソード、意味、手続き、コア、残存トレースの各メモリ。
- 保持は時間とともに減衰し、反復や想起によって強化されます。
- 証拠の状態は、顕著性や感情とは分離されたままです。
- 関連する想起は、ローカル GGUF モデルに渡す前に上限が設けられます。
- 弱く古い言語情報は、小さな「ゴーストトレース」に圧縮される場合があります。
- トレース記録はプロンプト想起から除外され、事実としては数えられません。
- コア／保護されたメモリは自動的にトレースされることはありません。
- 自動メンテナンスは決して完全削除（ハードデリート）しません。
- 明示的に忘却するには `FORGET-<memory_id>` が必要です。

これは心理学に着想を得た機能的なソフトウェアのアナロジーです。生物学的シミュレーションではなく、意識を確立するものでもありません。

## ビルド境界

この指令は、検証または GitHub の失敗があると停止します。モデルの重み、認証情報、`.secrets`、秘密鍵、ローカル データベースをコミットすることは決してありません。メッセージ送信、取引、送金、コンテンツ公開を行うことも決してありません。
