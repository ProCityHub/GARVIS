# GARVIS メモリライフサイクル ディレクティブ v1

**権限:** Adrien D. Thomas / ProCityHub  
**リポジトリ:** `~/GARVIS`  
**ブランチ:** `feature/memory-lifecycle-v1`

このディレクティブは、ローカルで証拠に配慮したメモリライフサイクルを導入し、その後、Git ワークフローを自動実行します: 検証、コミット、プッシュ、Pull Request、CI 監視、squash マージ、ブランチ削除、およびローカル `main` の更新。

## メモリ設計

- ワーキング、エピソード、セマンティック、手続き、コア、残余トレースの各メモリ。
- 保持は時間とともに減衰し、反復／想起によって強化されます。
- 証拠ステータスは顕著性や感情とは分離されたままです。
- 関連する想起はローカルの GGUF モデルに到達する前に制限されます。
- 弱く古い言語は小さな「ゴーストトレース」に圧縮される場合があります。
- トレース記録はプロンプトの想起から除外され、事実として数えることはできません。
- コア／保護されたメモリは自動的にトレースされることは決してありません。
- 自動メンテナンスはハード削除を決して行いません。
- 明示的な忘却には `FORGET-<memory_id>` が必要です。

これは心理学に基づく機能的なソフトウェアの類比です。生物学的シミュレーションではなく、意識を確立するものでもありません。

## ビルド境界

このディレクティブは、いかなる検証エラーや GitHub での失敗でも停止します。モデルの重み、認証情報、`.secrets`、秘密鍵、ローカルデータベースを決してコミットしません。メッセージ送信、取引、送金、コンテンツ公開を決して行いません。
