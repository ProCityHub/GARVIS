# GARVIS アトラクタメモリ プロトタイプ

著者: Adrien D. Thomas

ステータス: 決定論的ソフトウェア実験

科学的境界: 本プロトタイプは意識、感覚、同一性の持続、真の AGI、生物学的同等性、量子的検証、または物理法則を確立しません。

## 目的

Hopfield 風の連想メモリ構成要素を構築します。これにより:

1. 二極パターンを分散した相互作用重みに格納します。
2. 完全、部分的、またはノイズを含む手がかりを受け付けます。
3. 反復的に安定状態へ収束します。
4. 収束、復元精度、エネルギー、失敗を報告します。

ネットワークの重みがストレージ機構です。ストレージなしで記憶が存在すると表現してはなりません。

## 状態表現

格納されるパターンは -1 と +1 のみを含みます。

想起の手がかりは 0 を含んでもよく、これは消去または未知の値を表します。

すべてのパターンは同じ長さでなければなりません。

## 重みの構築

次元 n の格納パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

すべての対角成分は 0 でなければなりません。

想起はファイル名、データベースキー、パターン識別子、またはルックアップインデックスではなく、重み行列を使用しなければなりません。

## 安定化規則

決定論的非同期ニューロン更新を使用します。

各ニューロンについて:

field = sum(W[i][j] * state[j])

field が正ならニューロンを +1 に設定します。
field が負ならニューロンを -1 に設定します。
field が 0 の場合は現在の値を保持します。
field と現在値がともに 0 の場合は +1 を使用します。

1 回のスイープで全ニューロンを 1 度ずつ更新します。

完全な 1 スイープで変更が生じなかったとき、ネットワークは収束します。

有限の最大スイープ回数は必須です。

## エネルギー

完全に二極の状態について:

E = -0.5 * state_transpose * W * state

決定論的非同期の安定化中にエネルギーは増加すべきではありません。

これはソフトウェアの不変条件であり、物理的エネルギーの証拠ではありません。

## 必須測定項目

各想起結果は以下を報告しなければなりません:

- 最終状態;
- 収束/非収束;
- スイープ回数;
- 状態変更の総数;
- エネルギーの推移;
- 目標が与えられた場合の厳密一致可否;
- 目標が与えられた場合のハミング距離;
- 警告または失敗理由。

## 必須テスト

実装は以下をテストしなければなりません:

1. 空パターンの拒否。
2. 次元不一致の拒否。
3. 不正値の拒否。
4. 重みの対称性とゼロ対角。
5. 完全な格納手がかりからの厳密想起。
6. 部分的手がかりからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同一シードによる決定論的結果。
9. 有界な非収束の取り扱い。
10. 非同期エネルギーの非増加。
11. 偽アトラクタの報告。
12. 複数パターンでの容量と干渉。

成功、失敗、あいまいな想起はいずれも可視のままでなければなりません。

## ベースライン

本実験はアトラクタ想起を以下と比較しなければなりません:

1. 明示的な最尤近傍パターン照合ベースライン。
2. 学習された連想的安定化なしに、消去ビットを決定論的に充填するノーメモリのベースライン。

比較は、別の実験が事前登録されていない限り、記述的なものです。

## エンジニアリング境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールが実行してはならないこと:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセス実行;
- Git 操作;
- シークレットアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 統合

後続の統合では、アトラクタ出力をオプションのコンテキストとして提供する場合があります。

想起された内容には来歴と確信度を含め、決して無条件の真実として扱ってはなりません。

アトラクタメモリは、セッション履歴、リポジトリ証拠、科学的主張、承認、外部アクション権限から分離されたままにします。
