# GARVIS アトラクタメモリ プロトタイプ

著者: Adrien D. Thomas

ステータス: 決定的なソフトウェア実験

科学的境界: 本プロトタイプは、意識、感受性、同一性の持続、真の AGI、生物学的同等性、量子的検証、あるいは物理法則を確立するものではありません。

## 目的

次を満たす Hopfield 型の連想メモリ要素を構築します:

1. 双極パターンを分散した相互作用の重みに格納します。
2. 完全・部分的・ノイズを含む手がかりを受け付けます。
3. 反復的に安定状態へ収束します。
4. 収束、復元精度、エネルギー、失敗を報告します。

ネットワークの重みが記憶のメカニズムです。これを、保存領域なしに記憶が存在するかのように表現してはなりません。

## 状態表現

保存されるパターンは -1 と +1 のみを含みます。

想起の手がかりには、消去または不明な値を表す 0 を含めてもかまいません。

すべてのパターンは同一長でなければなりません。

## 重みの構成

次元 n の保存パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

すべての対角成分は 0 でなければなりません。

想起は、ファイル名、データベースキー、パターン識別子、検索インデックスではなく、重み行列を使用しなければなりません。

## 収束規則

決定的な非同期ニューロン更新を使用します。

各ニューロンについて:

field = sum(W[i][j] * state[j])

field が正のとき、ニューロンを +1 に設定します。
field が負のとき、ニューロンを -1 に設定します。
field が 0 のとき、現在の値を保持します。
field と現在の値がともに 0 のとき、+1 を使用します。

1 回のスイープで全ニューロンを 1 度ずつ更新します。

1 回の完全なスイープで変更が生じなければ収束と見なします。

有限の最大スイープ回数を必須とします。

## エネルギー

完全に双極の状態に対して:

E = -0.5 * state_transpose * W * state

決定的な非同期の収束過程ではエネルギーは増加しないはずです。

これはソフトウェア上の不変条件であり、物理的エネルギーの証拠ではありません。

## 必須の測定項目

各想起結果は以下を報告しなければなりません:

- 最終状態;
- 収束/非収束;
- スイープ回数;
- 状態変更の総数;
- エネルギーの推移;
- ターゲットが与えられた場合の完全一致;
- ターゲットが与えられた場合の Hamming 距離;
- 警告または失敗理由。

## 必須のテスト

実装は次をテストしなければなりません:

1. 空パターンの却下。
2. 次元不一致の却下。
3. 無効値の却下。
4. 重みの対称性と対角のゼロ。
5. 保存済みの完全手がかりからの厳密想起。
6. 部分的手がかりからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同一のシードを用いた決定的な結果。
9. 上限付き非収束の扱い。
10. 非同期におけるエネルギーの非増加。
11. 誤アトラクタの報告。
12. 複数パターンにおける容量と干渉。

成功・失敗・曖昧な想起はすべて可視のままでなければなりません。

## ベースライン

本実験はアトラクタ想起を以下と比較しなければなりません:

1. 明示的な最も近いパターンの照合ベースライン。
2. 学習による連想的な収束を用いず、消去ビットを決定的に補完する無記憶ベースライン。

別個の実験が事前登録されていない限り、比較は記述的なものに留まります。

## エンジニアリング上の境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールが行ってはならないこと:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセス実行;
- Git 操作;
- シークレットアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 連携

後の連携では、アトラクタの出力を任意のコンテキストとして提供する場合があります。

想起された内容には来歴と確信度を含め、決して鵜呑みにしてはなりません。

アトラクタメモリは、セッション履歴、リポジトリの証拠、科学的主張、承認、および外部アクションの権限から分離されたままです。
