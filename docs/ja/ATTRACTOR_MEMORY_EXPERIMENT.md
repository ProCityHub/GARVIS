# GARVIS アトラクタメモリ試作

著者: Adrien D. Thomas

ステータス: 決定的なソフトウェア実験

科学的境界: このプロトタイプは、意識、感受性、同一性の持続、真の AGI、生物学的同等性、量子的検証、または物理法則を確立するものではありません。

## 目的

Hopfield 風の連想メモリコンポーネントを構築します。これは次を満たします。

1. 双極パターンを分散した相互作用重みに格納します。
2. 完全、部分的、またはノイズを含む手掛かりを受け付けます。
3. 反復的に安定状態へ収束します。
4. 収束、復元精度、エネルギー、および失敗を報告します。

ネットワークの重みが記憶の格納機構です。これを、保存なしに記憶が存在すると表現してはなりません。

## 状態表現

保存されるパターンは -1 と +1 のみを含みます。

想起の手掛かりは 0 を含んでもよく、これは消去または不明な値を表します。

すべてのパターンは同じ長さでなければなりません。

## 重みの構築

次元が n の保存パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

すべての対角要素は 0 でなければなりません。

想起は、ファイル名、データベースキー、パターン識別子、またはルックアップインデックスではなく、重み行列を使用しなければなりません。

## 収束規則

決定的な非同期ニューロン更新を使用します。

各ニューロンについて:

field = sum(W[i][j] * state[j])

field が正ならニューロンを +1 に設定します。
field が負ならニューロンを -1 に設定します。
field が 0 の場合は現在の値を保持します。
field と現在値がともに 0 の場合は +1 を使用します。

1 回のスイープで全ニューロンを一度ずつ更新します。

完全な 1 スイープで変更が生じなければ、ネットワークは収束したとみなします。

スイープ回数の有限な上限は必須です。

## エネルギー

完全に双極の状態に対して:

E = -0.5 * state_transpose * W * state

決定的な非同期収束の間、エネルギーは増加しないはずです。

これはソフトウェア上の不変条件であり、物理的エネルギーの証拠ではありません。

## 必要な測定項目

各想起結果は次を報告しなければなりません:

- 最終状態;
- 収束/非収束;
- スイープ回数;
- 状態変更の総数;
- エネルギートレース;
- 目標が与えられた場合の厳密一致;
- 目標が与えられた場合のハミング距離;
- 警告または失敗理由。

## 必須テスト

実装は次をテストしなければなりません:

1. 空パターンの拒否。
2. 次元不整合の拒否。
3. 無効値の拒否。
4. 重みの対称性と対角 0。
5. 完全な保存手掛かりからの厳密想起。
6. 部分手掛かりからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同一シード使用時の決定的な結果。
9. 有界な非収束の取り扱い。
10. 非同期エネルギーの非増加。
11. 誤アトラクタの報告。
12. 複数パターンにおける容量と干渉。

成功、失敗、曖昧な想起はすべて可視のままにしておく必要があります。

## ベースライン

本実験は、アトラクタ想起を次と比較しなければなりません:

1. 明示的な最近傍パターン探索のベースライン。
2. 学習済みの連想的収束を用いず、消去ビットを決定的に埋めるノーメモリのベースライン。

比較は、別個の実験が事前登録されていない限り、記述的なものとします。

## エンジニアリング境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールは次を実行してはなりません:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセス実行;
- Git 操作;
- シークレットアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 統合

後の統合では、アトラクタの出力を任意のコンテキストとして提供する場合があります。

想起された内容は由来と信頼度を含み、決して無条件の真実として扱われてはなりません。

アトラクタメモリは、セッション履歴、リポジトリの証拠、科学的主張、承認、および外部アクション権限とは分離されたままです。
