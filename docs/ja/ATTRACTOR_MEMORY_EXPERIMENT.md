# GARVIS アトラクタメモリ プロトタイプ

著者: Adrien D. Thomas

状態: 決定論的なソフトウェア実験

科学的境界: このプロトタイプは、意識、感受性、同一性の持続、真の AGI、生物学的同等性、量子的検証、または物理法則を確立しません。

## 目的

Hopfield 方式の連想メモリ構成要素を構築します。要件:

1. 分散した相互作用重みに両極性パターンを格納します。
2. 完全・部分的・ノイズを含む手掛かりを受け付けます。
3. 反復的に安定状態へと落ち着きます。
4. 収束、復元精度、エネルギー、失敗を報告します。

ネットワークの重みが記憶の仕組みです。ストレージなしに記憶が存在すると記述してはなりません。

## 状態表現

格納されるパターンは -1 と +1 のみを含みます。

想起の手掛かりは 0 を含んでもよく、これは消去または未知の値を表します。

すべてのパターンは同じ長さでなければなりません。

## 重み行列の構築

次元が n の格納パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

すべての対角成分は 0 でなければなりません。

想起は、ファイル名、データベースキー、パターン識別子、参照インデックスではなく、重み行列を用いて行わなければなりません。

## 収束規則

決定論的な非同期ニューロン更新を使用します。

各ニューロンについて:

field = sum(W[i][j] * state[j])

field が正ならニューロンを +1 に設定します。
field が負ならニューロンを -1 に設定します。
field が 0 の場合は現在値を保持します。
field と現在値がともに 0 の場合は +1 を使用します。

1 回のスイープで各ニューロンを 1 度更新します。

完全な 1 回のスイープで変化がなければ、ネットワークは収束したと見なします。

有限の最大スイープ回数は必須です。

## エネルギー

完全に両極性の状態について:

E = -0.5 * state_transpose * W * state

エネルギーは、決定論的非同期の収束中に増加すべきではありません。

これはソフトウェアの不変条件であり、物理的なエネルギーの証拠ではありません。

## 必要な測定項目

各想起結果は次を報告しなければなりません:

- 最終状態;
- 収束/非収束;
- スイープ回数;
- 状態変更の合計数;
- エネルギーのトレース;
- 目標が与えられた場合の厳密一致;
- 目標が与えられた場合のハミング距離;
- 警告または失敗理由。

## 必要なテスト

実装は次をテストしなければなりません:

1. 空パターンの拒否。
2. 次元不整合の拒否。
3. 無効値の拒否。
4. 対称な重みとゼロ対角。
5. 完全な格納手掛かりからの厳密想起。
6. 部分手掛かりからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同じシードを用いた決定論的な結果。
9. 有界な非収束の取り扱い。
10. 非同期におけるエネルギーの非増加。
11. 誤アトラクタの報告。
12. 複数パターンにおける容量と干渉。

成功・失敗・曖昧な想起はいずれも可視のままでなければなりません。

## ベースライン

本実験は、アトラクタ想起を次と比較しなければなりません:

1. 明示的な最近傍パターン参照のベースライン。
2. 学習による連想的な収束を用いずに、消去ビットを決定論的に埋める「メモリなし」のベースライン。

別の実験が事前登録されない限り、この比較は記述的なものに留まります。

## エンジニアリング境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールが行ってはならないこと:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセスの実行;
- Git 操作;
- シークレットへのアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 統合

後の統合では、アトラクタの出力を任意のコンテキストとして提供する場合があります。

想起された内容には由来情報と信頼度を含め、決して疑いなく真実として扱ってはなりません。

アトラクタメモリは、セッション履歴、リポジトリのエビデンス、科学的主張、承認、外部アクションの権限から分離されたままです。
