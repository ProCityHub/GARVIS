# GARVIS アトラクター記憶 プロトタイプ

著者: Adrien D. Thomas

ステータス: 決定論的なソフトウェア実験

科学的境界: このプロトタイプは、意識、感受性、同一性の持続、真の AGI、生物学的同等性、量子的検証、または物理法則を確立しません。

## 目的

Hopfield 風の連想メモリ コンポーネントを構築します。要件:

1. 二極性パターンを分散した相互作用重みに格納します。
2. 完全、部分的、またはノイズを含む手がかりを受け付けます。
3. 反復的に安定状態へ収束します。
4. 収束、復元精度、エネルギー、失敗を報告します。

ネットワークの重みがストレージ機構です。ストレージなしにメモリが存在すると記述してはなりません。

## 状態表現

格納されるパターンは -1 と +1 のみを含みます。

想起の手がかりには、消去または未知の値を表す 0 も含められます。

すべてのパターンは同じ長さでなければなりません。

## 重みの構築

次元 n の格納パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

すべての対角成分は 0 でなければなりません。

想起は、ファイル名、データベースキー、パターン識別子、またはルックアップインデックスではなく、重み行列を使用しなければなりません。

## 安定化規則

決定論的な非同期ニューロン更新を使用します。

各ニューロンに対して:

field = sum(W[i][j] * state[j])

field が正ならニューロンを +1 に設定します。
field が負ならニューロンを -1 に設定します。
field が 0 の場合は現在の値を保持します。
field と現在の値がどちらも 0 の場合は +1 を使用します。

1 回のスイープで、すべてのニューロンを 1 回ずつ更新します。

完全な 1 回のスイープで変更が発生しなければ、ネットワークは収束します。

スイープの最大回数は有限で必須です。

## エネルギー

完全に二極性の状態に対して:

E = -0.5 * state_transpose * W * state

決定論的な非同期安定化の間、エネルギーは増加しないはずです。

これはソフトウェアの不変条件であり、物理的エネルギーの証拠ではありません。

## 必須測定

各想起結果は以下を報告しなければなりません:

- 最終状態;
- 収束または非収束;
- スイープ回数;
- 総状態変更数;
- エネルギー推移;
- 目標が与えられた場合の完全一致の有無;
- 目標が与えられた場合のハミング距離;
- 警告または失敗理由。

## 必須テスト

実装は次をテストしなければなりません:

1. 空パターンの拒否。
2. 次元不一致の拒否。
3. 無効値の拒否。
4. 対称な重みと対角成分 0。
5. 完全な格納手がかりからの厳密な想起。
6. 部分的手がかりからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同一シードを用いた決定論的結果。
9. 有界な非収束の取り扱い。
10. 非同期エネルギーの非増加。
11. 偽アトラクターの報告。
12. 複数パターンでの容量と干渉。

成功、失敗、曖昧な想起はすべて可視のままでなければなりません。

## ベースライン

本実験はアトラクター想起を次と比較しなければなりません:

1. 明示的な最近傍パターンのルックアップ・ベースライン。
2. 学習による連想安定化を用いずに、消去ビットを決定論的に埋めるメモリなしのベースライン。

比較は、別個の実験が事前登録されていない限り、記述的なものです。

## エンジニアリング境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールが実行してはならないこと:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセス実行;
- Git 操作;
- シークレットへのアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 統合

後続の統合では、アトラクターの出力を任意のコンテキストとして提供する場合があります。

想起された内容には来歴と信頼度を含め、断定的な真実として扱ってはなりません。

アトラクター記憶は、セッション履歴、リポジトリの証拠、科学的主張、承認、外部アクション権限とは分離されたままにします。
