# GARVIS アトラクタ記憶プロトタイプ

著者: Adrien D. Thomas

ステータス: 決定論的ソフトウェア実験

科学的境界: 本プロトタイプは、意識、感受性、同一性の持続、真の AGI、 生物学的同等性、量子的妥当性、または物理法則を確立するものではありません。

## 目的

Hopfield 型の連想記憶コンポーネントを構築します。要件は次のとおりです。

1. 分散した相互作用重みに双極パターンを保存する。
2. 完全・部分的・ノイズを含むキューを受け付ける。
3. 反復的に安定状態へ収束する。
4. 収束、復元精度、エネルギー、失敗を報告する。

ネットワークの重みが記憶の保存機構です。これを、保存なしに記憶が存在すると表現してはなりません。

## 状態表現

保存するパターンは -1 と +1 のみを含みます。

想起のキューは 0 を含んでもよく、これは消去または不明な値を表します。

すべてのパターンは同一の長さでなければなりません。

## 重みの構成

次元 n の保存パターン x_k に対して:

W = (1 / n) * sum of outer_product(x_k, x_k)

重み行列は対称でなければなりません。

対角成分はすべて 0 でなければなりません。

想起はファイル名、データベースキー、パターン識別子、ルックアップインデックスではなく、重み行列を用いなければなりません。

## 収束規則

決定論的非同期ニューロン更新を使用します。

各ニューロンについて:

field = sum(W[i][j] * state[j])

field が正ならニューロンを +1 に設定します。
field が負ならニューロンを -1 に設定します。
field が 0 の場合は現在値を保持します。
field と現在値がともに 0 の場合は +1 を使用します。

1 回のスイープで各ニューロンを 1 度更新します。

完全な 1 スイープで変更が生じなくなったとき、ネットワークは収束します。

スイープ回数の有限上限は必須です。

## エネルギー

完全に双極の状態に対して:

E = -0.5 * state_transpose * W * state

決定論的非同期の収束過程において、エネルギーは増加すべきではありません。

これはソフトウェアの不変条件であり、物理的エネルギーの証拠ではありません。

## 必要な測定項目

各想起結果は次を報告しなければなりません:

- 最終状態;
- 収束/非収束;
- スイープ回数;
- 総状態変更数;
- エネルギーの推移;
- ターゲットが与えられた場合の完全一致可否;
- ターゲットが与えられた場合のハミング距離;
- 警告または失敗理由。

## 必要なテスト

実装は次をテストしなければなりません:

1. 空パターンの拒否。
2. 次元不整合の拒否。
3. 不正値の拒否。
4. 対称な重みおよびゼロ対角。
5. 完全な保存キューからの厳密想起。
6. 部分的キューからの想起。
7. 制御されたビット反転ノイズからの想起。
8. 同一シードでの決定論的一致結果。
9. 境界付き非収束の取り扱い。
10. 非同期におけるエネルギー非増加。
11. 誤アトラクタの報告。
12. 複数パターン時の容量と干渉。

成功・失敗・曖昧な想起はすべて可視のままにする必要があります。

## ベースライン

実験はアトラクタ想起を次と比較しなければなりません:

1. 明示的な最近傍パターン照合のベースライン。
2. 学習による連想的な収束を用いず、消去ビットを決定的に埋める非記憶ベースライン。

別実験が事前登録されていない限り、比較は記述的なものに留めます。

## エンジニアリングの境界

最初の実装は Python 標準ライブラリのみを使用します。

提案モジュール:

src/garvis/attractor_memory.py

提案テスト:

tests/garvis/test_attractor_memory.py

メモリモジュールが実行してはならないこと:

- ファイルシステムへの書き込み;
- ネットワークアクセス;
- サブプロセス実行;
- Git 操作;
- シークレットへのアクセス;
- モデル呼び出し;
- 外部アクション。

## 将来の GARVIS 統合

後続の統合では、アトラクタの出力を任意のコンテキストとして提供する場合があります。

想起結果には出所と確信度を含め、無批判な真実として扱ってはなりません。

アトラクタ記憶は、セッション履歴、リポジトリの証拠、科学的主張、承認、および外部アクションの権限とは分離されたままです。
